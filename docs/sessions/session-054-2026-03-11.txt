   - Tick engine with configurable interval (60s/300s/900s/3600s)
   - scheduleTick/executeTick cycle: collect actions → AI compute → executeTurn → broadcast

   - Player join: find available AI kingdom → claim → send visible state
   - Player disconnect: mark disconnected, 5min grace → revert to AI
   - Season end: stop ticks, notify players, persist final state, cleanup
   - World list: getWorldList() with human count, turn, tick info
   - Persistence: serialize world meta + gameState to DB with "world:" prefix
   - Graceful shutdown: stop all timers, persist all worlds

ARCHITECTURE NOTES
- World map = single flat MapCell[] array (same as regular game, just larger)
- Kingdom tiles are composited at generation time, not at runtime
- All existing game logic (executeTurn, AI, combat, buildings, tech) works unchanged on world maps
- WorldServer is completely independent of GameManager — no shared state
- Tick timer uses setTimeout (not setInterval) for drift correction
- Global MAP_WIDTH/MAP_HEIGHT must be reconfigured before each tick (world dims may differ from classic games)
