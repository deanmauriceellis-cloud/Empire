Phase 13: Accounts & Persistence

## Summary
Implemented user accounts, JWT authentication, kingdom persistence in the database,
and client login/register UI. Players must now authenticate to access Kingdom World
mode. Kingdoms are recorded in the database and linked to user accounts for reconnection.

## Architecture Decisions
- JWT secret auto-generated at startup (override via JWT_SECRET env var)
- Access and refresh tokens are distinct (can't use one as the other)
- Username case-insensitive in DB (COLLATE NOCASE)
- Auth required only for world mode (join/reconnect), classic mode unchanged
- Kingdom records persist even when worlds are in memory
- Client stores tokens in localStorage for cross-session persistence
- Auth error on WS triggers token refresh attempt
