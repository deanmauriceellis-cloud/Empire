ARCHITECTURE NOTES
------------------
The AIPlanner is the core architectural change. It separates AI computation into
discrete steps that can be spread across time:

  Current:  [====SPIKE====]  idle idle idle  [====SPIKE====]
  New:      [==spread==across==tick==window==] [==spread==...==]

The planner processes units one at a time in MOVE_ORDER priority. Transport claim
sets (claimedUnitIds, claimedPickupLocs) are maintained incrementally across steps,
preserving the same sequential processing semantics as the original.

Key invariant: createAIPlanner + drain all steps === computeAITurn (same output,
same ordering, verified by determinism test).

WorldServer integration: planners start immediately after executeTick completes,
run in 50ms intervals processing 20 units per burst. By next tick, all planners
are typically done — actions are collected with near-zero cost at tick boundary.
