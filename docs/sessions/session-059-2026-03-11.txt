## Key Design Decisions
1. VIP bonus uses optional GameState.vipPlayers field — no game engine coupling to store module
2. Lazy Stripe import — server works without Stripe configured
3. Dev-mode granting — full store testable without payment processing
4. Per-category cosmetic equip — equipping auto-unequips same category
5. Late-binding setActions() on StorePanel — resolves circular dependency in main.ts
