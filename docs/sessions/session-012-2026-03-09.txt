## Decisions & Rationale
- Decision: Single-stage Dockerfile (no multi-stage) — Rationale: Server uses `tsx` for runtime TS transpilation, so we need dev dependencies at runtime. A multi-stage build would save ~50MB but add complexity for marginal gain.
- Decision: Express v5 `/{*splat}` instead of `*` — Rationale: Express v5 uses path-to-regexp v8 which requires named wildcard parameters. Bare `*` throws `PathError`.
- Decision: SPA fallback as catch-all — Rationale: Client is a SPA, any non-API/non-static route should serve index.html for client-side routing.
- Decision: Archive sessions 004-007 from CHANGELOG — Rationale: CHANGELOG exceeded 200-line threshold (was 256 lines).
