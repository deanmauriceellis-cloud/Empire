## Changes Made
### 1. Map Generation Module (mapgen.ts)
- Seedable PRNG using mulberry32 algorithm — fast, deterministic, good distribution
- Height map generation: random heights 0-998, 9-point neighborhood averaging with configurable smoothing passes (default 5)
- Waterline calculation: histogram-based threshold to achieve target water ratio (default 70%)
- Terrain assignment: land/sea based on waterline, edge cells marked off-board
- City placement: random land cell selection with adaptive minimum distance (sqrt(landArea/numCities)), distance decrements when candidates exhausted
- Continent detection: BFS flood-fill finding all connected land regions
- Continent scoring: (shore_cities*3 + inland_cities*2) * 1000 + area — matches original formula
- Starting city selection: scores continents, creates balanced pairs, picks most balanced pair
- Fallback for <2 viable continents: samples random distant city pairs
- Integrated `generateMap(config)` orchestrating all steps

### 2. Test Suite (mapgen.test.ts)
- 27 tests covering all components:
  - RNG: determinism, different seeds, value range
  - Height map: correct size, determinism, non-negative values, smoothing reduces variance
  - Terrain: water ratio within ±15% of target, edge cells off-board, interior cells on-board
  - Cities: correct count, all on board, City terrain type, all unowned, no duplicates
  - Continents: at least one found, every land cell in exactly one, every city assigned
  - Starting cities: two distinct indices, on different continents when possible
  - Integration: determinism, different seeds differ, correct city count, valid starting cities, valid terrain, consistent cityId references, works with various water ratios

## Decisions & Rationale
- Decision: Used mulberry32 PRNG — Rationale: Simple, fast, excellent distribution, deterministic from seed, no external dependency
- Decision: 9-point averaging for smoothing (not Perlin/simplex) — Rationale: Matches original C algorithm exactly, produces similar continent shapes
- Decision: Histogram-based waterline — Rationale: Matches original, more predictable than percentile-based
- Decision: Adaptive distance reduction for city placement — Rationale: Matches original, guarantees all NUM_CITY cities get placed even on cramped maps
- Decision: BFS flood-fill for continents — Rationale: Matches original, simple and correct
- Decision: Balanced pair selection for starting cities — Rationale: Provides fairest starting positions by default
