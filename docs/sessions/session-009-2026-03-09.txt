## Changes Made

### 1. HTML/CSS Overlay UI System
- Created `ui/styles.ts` with full CSS injected via `<style>` tag
- Dark theme with semi-transparent panels, monospace font, blue/gold accents
- `pointer-events: none` wrapper with `pointer-events: auto` on interactive elements
- Scrollbar styling, button hover/active states, modal overlay

### 2. HUD (Top Bar + Bottom Bar)
- Top bar: Turn number, player city count, unit count, version
- Bottom bar: Selected unit (name, HP, moves, behavior, cargo) or city (production, progress, turns remaining)
- Falls back to "Click a unit or city to select" when nothing selected

### 3. Minimap
- Dedicated 200x120 `<canvas>` (2px per tile for 100x60 map)
- Color-coded: land (green), sea (blue), fog (black), cities (blue/red/gray), units (bright dots)
- White viewport rectangle showing current camera view
- Click-to-navigate: click on minimap pans camera to that tile

### 4. Click-to-Move Interaction
- `moveCalc.ts`: computes valid adjacent moves and attacks for selected unit
- `highlights.ts`: sprite-pooled tile highlight renderer with pulse animation
- Green diamonds for valid moves, red diamonds for attack targets
- Click highlighted tile to execute move/attack; auto-advance when out of moves
- Right-click to deselect

### 5. Action Panel
- Context-sensitive buttons based on selection
- Unit selected: Skip (Space), Sentry (G), Explore (F), Wait for Transport (T), Disembark (U)
- City selected: Change Production (P)
- Always shown: Next Unit (N), End Turn (Enter)
- Buttons grayed out when invalid

### 6. City Production Panel
- Modal overlay with 3x3 grid of 9 unit types
- Shows unit char, name, build time, attack strength, max HP
- Current production highlighted, progress bar with percentage
- 20% switch penalty warning appears on hover over different production
- Close via X button or Escape key

### 7. Turn Flow Manager
- Scans for units needing orders (func=None, has moves, not embarked)
- Auto-cycles through them, panning camera to each
- Skip unit (keeps in pool for later) vs mark done (permanent)
- Next Unit / End Turn buttons

### 8. Event Log
- Scrollable panel (bottom-left, max 30 events)
- Color-coded: combat=orange, capture=gold, production=green, death=gray, discovery=blue
- Click event to pan camera to its location

### 9. Menu Screens
- Main menu: "EMPIRE REBORN" title, version subtitle, New Game button
- Game over: Victory/Defeat text, final stats (turn, cities, units), New Game button

### 10. Action Collector
- Accumulates PlayerActions during a turn
- Applies actions immediately to GameState via `processAction` for visual feedback
- Updates vision via `scan` after each move
- Tracks moved unit IDs for turn flow integration
- `endTurn()` submits batch to `game.submitTurn(actions)` for AI + production + repair

### 11. Input System Upgrade
- Added event queues alongside existing polling
- `consumeClicks()`: left-click events with position + shiftKey
- `consumeRightClicks()`: right-click events (context menu prevented)
- `consumeKeyPresses()`: one-shot key events (not held)
- Existing `isKeyDown()` kept for camera panning (continuous)

## Decisions & Rationale
- Decision: HTML/CSS overlay instead of PixiJS UI — Rationale: text-heavy panels need CSS fonts, flexbox, scrolling, hover states; pointer-events passthrough keeps canvas interaction clean
- Decision: Click-to-move instead of keyboard directions — Rationale: user explicitly preferred modern UI; eliminates WASD/movement key conflict; more intuitive for web game
- Decision: Apply actions immediately via processAction — Rationale: gives instant visual feedback; moves are no-ops when re-processed by executeTurn; repair tracking uses action list not state
- Decision: Single camera instance (never recreated) — Rationale: UI components capture camera reference; recreating would leave stale references in minimap/eventLog
- Decision: G for sentry instead of S — Rationale: S is used for camera panning (WASD); G is for "guard"
- Decision: Shift+click city to open production — Rationale: regular click selects city, need modifier for modal; also accessible via P key or action panel button
