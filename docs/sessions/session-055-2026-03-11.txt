### Shield Mechanic
- Added `ShieldState` interface to types.ts: `{ chargeMs, activatedAt, isActive }`

- Added `isShielded()` helper in game.ts — checks if player has active shield
- Combat immunity in game.ts processAction:
  - Attack on shielded units: returns empty events, no damage
  - Attack on shielded cities: returns empty events, no capture
  - Bombard on shielded units/structures: returns empty events
  - autoAttackStructures: skips shielded enemy units
  - checkMineTrigger: shielded units don't trigger mines
- Shielded player can still attack others (shield is defensive only)

### Shield Lifecycle (WorldServer.ts)
- New player join: initializes shield with SHIELD_INITIAL_MS (2hr) charge
- Disconnect: accumulates charge from online time (onlineMs * SHIELD_CHARGE_RATIO), activates shield
- Grace timer uses shield charge duration (not fixed 5min)
- Shield expiry: player reverts to AI, shield charge zeroed
- Reconnect: deactivates shield, preserves remaining charge (chargeMs - elapsed)
- `playerConnectedAt` map tracks connection timestamps for charge calculation

### Reconnection (WorldServer.ts)
- New `handleReconnectWorld()` method (~50 lines)
- Validates: world exists, player exists, player not defeated
- Restores human control (isAI = false), clears disconnect timer
- Deactivates shield preserving remaining charge
- Sends world_joined + visible state to reconnecting client
