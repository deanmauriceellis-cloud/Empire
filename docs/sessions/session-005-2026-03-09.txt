## Changes Made

### 1. AI Production Strategy (Step 4.1)
- 4 ratio tables matching original: RATIO_1 (≤10 cities), RATIO_2 (11-20), RATIO_3 (21-30), RATIO_4 (>30)
- Hierarchical decision: continent defense → transport production → ratio balancing
- Lake city detection via continent BFS — lake cities only produce armies or fighters
- `overproduced()` checks if a type exceeds 1.5x its target ratio
- `needMore()` finds the unit type with greatest deficit vs target ratio
- Never builds carriers or satellites (matching original AI behavior)

### 2. AI Army Movement (Step 4.2)
- Adjacent attack priority using `findAdjacentAttack()` with view map character conversion
- `attackListToViewChars()` maps original attack strings to our view map conventions (enemy=lowercase, 'X'=enemy city, '*'=unowned)
- Land objective pathfinding via `armyFightMoveInfo()` — targets enemies, cities, exploration (11x weight)
- Cross-cost evaluation: compares land fight distance vs transport boarding distance
- `moveAway()` forces departure from cities when stuck
- `createTempViewMap()` marks loading transports as '$' for army boarding pathfinding

### 3. AI Transport Movement (Step 4.3)
- Two-state loading/unloading system based on cargo count
- Loading mode: seek armies via `createTTLoadViewMap()`, or explore if none found
- Unloading mode: navigate to enemy coasts via `createUnloadViewMap()` with continent value weighting (0-9)
- `shouldUnload()` — start unloading when >50% full and near enemy territory
- `tryUnloadArmies()` — disembark cargo onto adjacent land
- `tryLoadArmies()` — pick up armies at same location

### 4. AI Fighter Movement (Step 4.4)
- Attack adjacent enemies first (FIGHTER_ATTACK priority)
- Fuel management: return to base when range ≤ nearest city distance + 2
- `findNearestCityDist()` Chebyshev distance to nearest owned city
- Falls back to carrier if no city reachable
- Explores when no targets in range

### 5. AI Ship Movement (Step 4.4 cont.)
- Damage check first: if hits < maxHits, navigate to port ('O')
- Stay in port when damaged (return empty actions)
- Attack adjacent enemies when fully repaired (SHIP_ATTACK priority)
- Explore/patrol when no targets (21x exploration weight)
- Shared logic for all 5 ship types (patrol, destroyer, submarine, carrier, battleship)

### 6. AI Turn Orchestrator (Step 4.5)
- `computeAITurn(state, aiOwner)` returns PlayerAction[]
- Refresh vision: scan all AI pieces and cities
- Production decisions first
- Movement in MOVE_ORDER priority (satellite skipped — automatic)
- Surrender check: resign when < 1/3 enemy cities AND armies, or eliminated

## Decisions & Rationale
- Decision: Adapt attack lists via character conversion rather than redefining constants — Rationale: Keeps units.ts attack strings matching original source for reference, while cleanly adapting to our view map encoding
- Decision: Never build carriers or satellites — Rationale: Matches original AI behavior; carrier strategy was intentionally disabled in compmove.c
- Decision: Shared ship movement logic for all 5 ship types — Rationale: Original had the same function for all ships; differentiation happens through unit attributes (speed, hits, strength)
- Decision: 1.5x overshoot tolerance for production rebalancing — Rationale: Prevents constant switching; original used a similar threshold via discrete counting
- Decision: Cross-cost of 30 for army boarding — Rationale: Moderate bias toward staying on land; original used dynamic cross costs (14-60) based on target type
